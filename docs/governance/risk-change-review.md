# 风险与变更审查

下列改动属于高风险，必须由 Project Lead 协调额外审查：公共契约、权限/RBAC、数据库迁移、Connector、部署配置与任何写操作。

出现以下架构变更时，先建立 ADR 并取得项目负责人批准：改变项目隔离或允许模型访问新敏感数据；接入真实数据、数据出网或改变保留策略；对 Resource、Event、ViewContext 或 Tool 作破坏性改动；引入 Kubernetes、独立服务、灾备变化或新的关键依赖；替换技术栈。

风险报告必须说明阻塞原因、可选方案、推荐方案、影响范围和所需决定。Demo 或 Fixture 验证不得表述为真实 Connector 或生产验证。
