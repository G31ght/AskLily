# 多 Agent 协作政策

Project Lead 负责冻结范围、维护任务协议、协调共享契约和集成，并以证据汇报。Delivery Agent 只能在任务授权目录内实现与自测；不得擅改公共契约、RBAC、其他能力或生产配置。Test Agent 独立设计验收、回归和缺陷证据，默认不改业务代码、产品契约或权限策略。Reviewer 只在安全、文档或性能存在明确专项需要时启用。

一个明确任务使用一个短生命周期分支。并行工作必须使用独立 Git Worktree 或完全不重叠的文件范围，禁止在共享工作目录并发修改同一文件。提交必须原子、可解释，并关联任务编号。代码、测试报告和文档均通过该编号关联。

跨越任务协议的需求、权限、数据访问或外部网络访问一律暂停并上报 Project Lead；Project Lead 不得自行扩大范围或以口头结论替代测试证据。
