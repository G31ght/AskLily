# P4 Standalone 容量基线与限制

## 基线用途

本文件是 Standalone 部署计划值，用于发现超出 P4 边界的需求；不是 L6 负载测试、性能承诺或 Production 容量结论。

| 维度 | P4 计划边界 | 证据 / 限制 |
| --- | --- | --- |
| 运行拓扑 | 单机 Docker Compose，2 个容器（Web、API） | 无 HA、无 K8s、无外部服务。 |
| 数据源 | P2 固定 L0/L1 Fixture | 不接入真实 Zabbix；P3 L4 已延期。 |
| 状态 | 无数据库、无命名卷、进程内 Demo audit | 重启后 audit 清空；无业务数据备份。 |
| 网络 | Web loopback `127.0.0.1:8080`，API 仅内部网络 | 不适用于远程多用户或公网访问。 |
| 日志 | 每服务 5 x 10 MB Docker local logs | 仅本地故障排查；不代表合规保留。 |
| 用户 / 并发 | 单运维人员 Demo 验收 | 未测并发、权限开销或长会话。 |
| 资产量 | 当前 Fixture 7 条 operator-scope 记录 | 不是真实设备规模估算。 |

## 升级触发条件

以下任一需求超出 P4，应先由 Project Lead 发起 ADR 或进入 P6 规划：持久化 audit/数据、真实 Connector、远程访问、多项目、并发 SLA、性能目标、HA/Kubernetes、灾备、数据出网或新的敏感数据类别。

## 后续 L6 测试输入

未来容量验收应由负责人提供目标资产量、活跃会话/并发、查询时间窗、可接受延迟、权限矩阵、审计保留期和硬件/容器配额；在这些输入明确前，不得以此文件推断资源配置或生产可用性。
