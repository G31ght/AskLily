# 协作约束

本仓库遵循工程开工文档 v0.1。`main` 只承载当前可运行、可测试的已验收基线；保护规则生效后不得直接推送。

每项工作必须先有可验收任务，使用短生命周期分支和原子提交。多 Agent 只能使用独立 Git Worktree，或拥有明确且不重叠的文件范围；公共契约、RBAC、部署配置、数据库迁移、Connector、写操作及同一文件的修改由 Project Lead 串行协调。

开发完成仅代表自测完成。合并前必须附上任务编号、验证命令及结果；完整任务还需独立 Test Agent 的证据。任何新增真实数据访问、网络外发、密钥、写操作、权限或架构变更必须停止施工并升级给 Project Lead；需要时先记录 ADR 并取得项目负责人批准。
